## 1. Стек

| Технология       | Назначение                         | Версия / Примечание             |
| ---------------- | ---------------------------------- | ------------------------------- |
| React            | UI-фреймворк                       | 19+                             |
| TypeScript       | Типизация                          |                                 |
| Vite             | Сборка, dev-сервер                 |                                 |
| Tailwind CSS     | Стили                              | v4                              |
| Zustand          | Глобальный клиентский стейт        |                                 |
| TanStack Query   | Серверный стейт, кэш, инвалидация  |                                 |
| CodeMirror 6     | Markdown-редактор с live preview   |                                 |

**Языки интерфейса:** русский и английский. i18n-библиотека не нужна — строки хранятся в константах (`src/i18n/ru.ts` и `src/i18n/en.ts`). Все пользовательские строки обязаны присутствовать в обоих файлах; если ключ отсутствует в активном языке — используется русская строка как fallback.

**Дизайн:** тёмная тема, ориентир — VS Code. Только desktop (адаптивность не требуется).

## 2. Общие требования

- Приложение — SPA. Сервер отдаёт один `index.html`, роутинг на клиенте.
- Шрифты, цвета, отступы — задаются через Tailwind config (`theme.extend`), не хардкодятся в компонентах.
- Все обращения к API — через единый HTTP-клиент (`src/api/client.ts`). Клиент автоматически подставляет `Authorization: Bearer <token>` и обрабатывает 401 (пробует обновить токен через refresh; если refresh провалился — редирект на `/login`).
- Access token хранится в памяти (Zustand `authStore`), не в localStorage.
- При обновлении страницы — автоматический silent refresh через cookie (запрос к `POST /api/auth/refresh` выполняется до монтирования приложения).
- Ошибки сети отображаются пользователю (toast / inline — уточнено по разделам).
- Все временны́е метки приходят в UTC, отображаются в локальном времени пользователя.
- Все пользовательские строки (подписи, placeholder-ы, сообщения об ошибках, диалоги) берутся из активного i18n-файла. Хардкодить строки в компонентах запрещено. Строки в данном документе приведены на русском как пример — приложение отображает их в активном языке.

## 3. Роутинг

| Путь                        | Компонент             | Layout           | Описание                                   |
| --------------------------- | --------------------- | ---------------- | ------------------------------------------ |
| `/login`                    | `LoginPage`           | `AuthLayout`     | Страница входа                             |
| `/register`                 | `RegisterPage`        | `AuthLayout`     | Страница регистрации                       |
| `/confirm-email`            | `ConfirmEmailPage`    | `AuthLayout`     | Подтверждение email по ссылке из письма    |
| `/reset-password`           | `ResetPasswordPage`   | `AuthLayout`     | Запрос сброса или установка нового пароля  |
| `/`                         | `AppPage`             | `AppLayout`      | Основное приложение                        |
| `*`                         | `NotFoundPage`        | —                | 404                                        |

Неаутентифицированный пользователь, обращающийся к `/`, перенаправляется на `/login`.  
Аутентифицированный пользователь, обращающийся к `/login` или `/register`, перенаправляется на `/`.

**404-страница (`NotFoundPage`):** полноэкранный экран без `AppLayout` и `AuthLayout`. По центру — лаконичное сообщение «Страница не найдена» + кнопка «На главную» → `/`. Аутентификация не требуется для отображения.

## 4. Основной layout

`AppLayout` — корневой layout для аутентифицированных пользователей. Делится на три вертикальные панели:

```
┌──────┬────────────────────────┬──────────────────┐
│      │                        │                  │
│ Left │       Middle           │   Right (Chat)   │
│      │                        │   (скрываемая)   │
│      │                        │                  │
└──────┴────────────────────────┴──────────────────┘
```

- Левая панель — фиксированная ширина, скрываемая.
- Средняя панель — занимает всё оставшееся пространство (`flex: 1`).
- Правая панель — фиксированная ширина, скрываемая (toggle через кнопку «Чат» на левой панели).

---

### 4.1 Левая панель

Узкая вертикальная панель-навигатор (аналог Activity Bar в VS Code). Содержит только иконки с tooltip-подсказками при наведении.

**Элементы (сверху вниз):**

| Элемент          | Действие                                                                 | Примечание                                                        |
| ---------------- | ------------------------------------------------------------------------ | ----------------------------------------------------------------- |
| Логотип          | Не кликабелен                                                            | Декоративный элемент                                              |
| Все заметки      | Открывает вкладку «Все заметки» в средней панели                        |                                                                   |
| Поиск            | Открывает вкладку «Поиск» в средней панели                              |                                                                   |
| Теги             | Открывает вкладку «Теги» в средней панели                               |                                                                   |
| Чат              | Toggle правой панели (открыть / закрыть)                                | Активное состояние — иконка подсвечена; бейдж с числом `pending` proposals поверх иконки (скрывается при 0) |
| *(разделитель)*  |                                                                          |                                                                   |
| Корзина          | Открывает вкладку «Корзина» в средней панели                            |                                                                   |
| *(разделитель)*  |                                                                          | Нижняя группа прижата к низу                                      |
| Тема             | Переключает тему (тёмная ↔ светлая)                                     | Иконка солнца / луны; выбор персистируется в `localStorage`       |
| Профиль          | Открывает попап профиля                                                  | Иконка: аватар или инициал из никнейма                            |
| Админ-панель     | Открывает вкладку «Админ-панель» в средней панели                       | Видна только пользователям с ролью `admin`                        |

**Состояния кнопок:**
- Default, Hover, Active (текущая вкладка открыта в средней панели).
- Кнопка «Чат» имеет отдельное активное состояние (панель открыта).

---

### 4.2 Средняя панель (вкладки)

Tabbed-контейнер — аналог редактора вкладок VS Code.

**Таб-бар:**
- Расположен в верхней части средней панели.
- Каждая вкладка содержит: иконку (опционально), заголовок, кнопку закрытия `×`.
- Вкладки можно закрывать. Системные вкладки (Все заметки, Поиск и т.д.) закрываются так же, как редакторские.
- При закрытии последней вкладки — отображается пустой экран-заглушка с подсказкой: «Откройте заметку или выберите раздел на боковой панели».
- Несохранённые изменения в редакторе — индикатор (точка на вкладке, аналог VS Code).
- При закрытии вкладки редактора с несохранёнными изменениями — диалог подтверждения: «Закрыть без сохранения? Несохранённые изменения будут потеряны.» Кнопки: «Закрыть» (закрыть без сохранения) и «Отмена». Если автосохранение в процессе — дождаться его завершения перед закрытием.

**Типы вкладок:**

| ID типа       | Заголовок вкладки                   | Описание                            |
| ------------- | ----------------------------------- | ----------------------------------- |
| `all-notes`   | «Все заметки»                       | Список заметок пользователя         |
| `search`      | «Поиск»                             | Поиск по заметкам                   |
| `tags`        | «Теги»                              | Управление тегами                   |
| `trash`       | «Корзина»                           | Удалённые заметки                   |
| `note:{id}`   | Заголовок заметки / «Без названия»  | Редактор конкретной заметки         |
| `admin`       | «Администрирование»                 | Управление пользователями           |

Системных вкладок (`all-notes`, `search`, `tags`, `trash`, `admin`) не может быть более одной каждого типа одновременно. При повторном нажатии на кнопку левой панели — фокус переходит на уже открытую вкладку.

## 5. Страницы аутентификации

Все auth-страницы используют `AuthLayout` — центрированная карточка (max-width: 420px) на тёмном фоне. Карточка содержит логотип приложения вверху, затем заголовок страницы, затем форму.

---

### 5.1 Логин

**Поля:**

| Поле     | Тип      | Валидация                              |
| -------- | -------- | -------------------------------------- |
| Email    | `email`  | Обязательное; базовый формат email     |
| Пароль   | `password` | Обязательное; минимум 1 символ (настоящая проверка на сервере) |

**Поведение:**
- Валидация запускается при `blur` на полях и при сабмите.
- При сабмите — кнопка «Войти» переходит в состояние загрузки (`disabled` + spinner).
- **Успех:** access token сохраняется в `authStore`, redirect на `/`.
- **Ошибка 401:** inline-сообщение под формой: «Неверный email или пароль».
- **Ошибка 403 (email не подтверждён):** inline-сообщение: «Email не подтверждён. Проверьте почту или запросите новое письмо». Под сообщением — кнопка «Отправить повторно» (`POST /api/auth/resend-confirmation`). Кнопка имеет cooldown 60 секунд после нажатия.
- **Ошибка 403 (аккаунт заблокирован):** inline-сообщение: «Ваш аккаунт заблокирован. Обратитесь к администратору» (код `account_blocked` в теле ответа).
- **Ошибка сети:** toast «Ошибка соединения».
- **Rate limit 429:** inline-сообщение: «Слишком много попыток. Подождите немного».

**Ссылки:**
- «Забыли пароль?» → `/reset-password`
- «Нет аккаунта? Зарегистрироваться» → `/register`

---

### 5.2 Регистрация

**Поля:**

| Поле              | Тип        | Валидация (клиентская)                                                                 |
| ----------------- | ---------- | -------------------------------------------------------------------------------------- |
| Email             | `email`    | Обязательное; базовый формат                                                           |
| Никнейм           | `text`     | Обязательное; 2–32 символа; только латиница, кириллица, цифры, `_`, `-`               |
| Пароль            | `password` | Обязательное; ≥8 символов; ≥1 заглавная, ≥1 строчная, ≥1 цифра                        |
| Повтор пароля     | `password` | Обязательное; совпадает с полем «Пароль»                                               |

Требования к паролю отображаются в виде чеклиста под полем по мере ввода (✓/✗ для каждого правила).

**Поведение:**
- Валидация при `blur` + при сабмите.
- При сабмите — кнопка «Зарегистрироваться» переходит в состояние загрузки.
- **Успех:** карточка меняется на экран подтверждения: «Мы отправили письмо на `{email}`. Перейдите по ссылке для активации аккаунта». Кнопка «Отправить повторно» с cooldown 60 секунд.
- **Ошибка 409 (email занят):** inline под полем email: «Этот email уже зарегистрирован».
- **Ошибка 409 (никнейм занят):** inline под полем никнейма: «Этот никнейм уже занят».
- **Ошибка сети:** toast «Ошибка соединения».

**Ссылки:**
- «Уже есть аккаунт? Войти» → `/login`

---

### 5.3 Подтверждение email

Страница открывается по ссылке из письма. URL содержит `?token=...`.

**Поведение:**

| Состояние                        | Отображение                                                                                       |
| -------------------------------- | ------------------------------------------------------------------------------------------------- |
| Загрузка (отправляется запрос)   | Spinner по центру карточки + текст «Подтверждаем email...»                                        |
| Успех                            | Иконка ✓ + «Email подтверждён! Теперь вы можете войти.» + кнопка «Войти» → `/login`              |
| Ошибка: токен истёк              | Иконка ✗ + «Ссылка устарела. Запросите новую.» + поле email + кнопка «Отправить новую ссылку»    |
| Ошибка: токен не найден / неверный | Иконка ✗ + «Ссылка недействительна.» + кнопка «На страницу входа» → `/login`                  |
| Ошибка сети                      | Toast «Ошибка соединения» + кнопка «Попробовать снова»                                            |

- Запрос к `POST /api/auth/confirm-email` с `{ token }` выполняется автоматически при монтировании страницы.
- Повторная отправка ссылки: `POST /api/auth/resend-confirmation` с `{ email }`. После успеха — «Новое письмо отправлено».

### 5.4 Сброс пароля

Страница `/reset-password` работает в двух режимах в зависимости от наличия `?token=...` в URL.

#### Шаг 1 — Запрос сброса (без токена)

**Поле:** Email (обязательное, базовый формат).

**Поведение:**
- Кнопка «Сбросить пароль» → `POST /api/auth/request-password-reset`.
- **Успех:** форма заменяется сообщением: «Если аккаунт с таким email существует, мы отправили письмо со ссылкой для сброса пароля.» (ответ намеренно одинаков для существующего и несуществующего email — защита от энумерации). Кнопка «Назад к входу» → `/login`.
- **Ошибка 429:** «Слишком много запросов. Подождите немного.»
- **Ошибка сети:** toast «Ошибка соединения».

#### Шаг 2 — Установка нового пароля (с токеном)

**Поля:**

| Поле          | Тип        | Валидация                                                      |
| ------------- | ---------- | -------------------------------------------------------------- |
| Новый пароль  | `password` | Обязательное; ≥8 символов; ≥1 заглавная, ≥1 строчная, ≥1 цифра |
| Повтор пароля | `password` | Совпадает с полем «Новый пароль»                               |

Требования к паролю отображаются чеклистом под полем.

**Поведение:**
- Кнопка «Сохранить новый пароль» → `POST /api/auth/reset-password` с `{ token, new_password }`.
- **Успех:** «Пароль успешно изменён.» + кнопка «Войти» → `/login`.
- **Ошибка 400/404 (токен истёк или неверный):** «Ссылка для сброса пароля недействительна или устарела. Запросите новую.» + кнопка «Запросить снова» → `/reset-password` (без токена).
- **Ошибка сети:** toast «Ошибка соединения».

## 6. Вкладки средней панели

### 6.1 Все заметки

Список всех активных (не удалённых) заметок пользователя.

**Строка заметки:**

```
┌──────────────────────────────────────────────────────────────────┐
│ Заголовок / «Без названия»   [тег1] [тег2]     12 апр, 14:32    │
│ Начало содержимого заметки (preview, ~1–2 строки)...             │
└──────────────────────────────────────────────────────────────────┘
```

- Слева: заголовок (жирный) + первые строки содержимого (приглушённый цвет).
- Справа: теги заметки (chips) + дата последнего редактирования.
- Клик по строке → открывает вкладку `note:{id}` в средней панели.

**Прокрутка:** бесконечный скролл (подгрузка следующей страницы при приближении к концу списка).

**Фильтрация по тегу:**
- Выпадающий список / панель тегов над списком заметок.
- Можно выбрать один или несколько тегов — показываются заметки, имеющие **все** выбранные теги (AND).
- Отдельный пункт «Без тегов» — показывает заметки без единого тега.

**Удаление:**
- Одиночное: кнопка удаления на строке (появляется при hover) или через контекстное меню.
- Множественное: чекбоксы на строках при hover → появляется панель действий с кнопкой «Удалить выбранные (N)».
- Удаление — soft delete (заметки уходят в корзину).

**Кнопка «Создать заметку»:**
- Расположена в верхней части вкладки.
- Создаёт пустую заметку → сразу открывает вкладку редактора `note:{id}`.

---

### 6.2 Поиск

**Поле ввода:** одно поле с иконкой поиска. Запрос выполняется при отправке (Enter или кнопка). Строка и результаты сохраняются при переключении на другие вкладки и восстанавливаются при возврате.

**Два параллельных поиска при каждом запросе:**

| Тип                | API                              | Описание                                                   |
| ------------------ | -------------------------------- | ---------------------------------------------------------- |
| По ключевым словам | `GET /api/notes?search={query}`  | Полнотекстовый поиск по содержимому и заголовку            |
| Семантический      | `POST /api/rag/search`           | Векторный поиск по чанкам; возвращает score похожести      |

**Отображение результатов — два раздела последовательно:**

1. **«По ключевым словам»** — список заметок с превью совпадающего фрагмента (совпадение выделено).
2. **«Семантический поиск»** — список заметок с процентом похожести.

В семантическом разделе: несколько чанков одной заметки **визуально объединяются** в одну карточку. Карточка показывает заголовок заметки + все совпавшие чанки (превью каждого) + наибольший score среди чанков.

**Клик по результату** → открывает вкладку `note:{id}` в средней панели.

**Состояния:**
- Загрузка обоих поисков: skeleton-плейсхолдеры в обоих разделах.
- Один из поисков завершился раньше: показывается его результат, другой продолжает загрузку.
- Пустые результаты в разделе: «Ничего не найдено» (каждый раздел независимо).
- Ошибка раздела: «Не удалось выполнить поиск» + кнопка «Повторить».

---

### 6.3 Теги

**Список тегов:** каждая строка содержит имя тега и количество заметок с этим тегом.

**Действия:**
- **Создать тег** — поле ввода + кнопка (inline-форма) в верхней части вкладки. Отправка по Enter или кнопке. Валидация: 1–50 символов. Дублирующее имя (без учёта регистра) — inline-ошибка под полем: «Тег с таким именем уже существует».
- **Переименовать** — inline-редактирование при двойном клике или через иконку карандаша на строке. Сохранение по Enter или потере фокуса. Отмена по Escape.
- **Удалить** — кнопка на строке (hover). Перед удалением — диалог подтверждения: «Тег «{name}» будет отвязан от {N} заметок. Продолжить?».

**Валидация** соответствует CPS (1–50 символов, уникальность регистронезависимая).

**Состояния:** загрузка (skeleton), пустой список («Тегов пока нет. Создайте первый тег.»), ошибка загрузки.

---

### 6.4 Корзина

Список удалённых заметок, отсортированный по дате удаления (новые сверху).

**Строка заметки:** аналогична «Все заметки» + индикатор оставшегося времени хранения («Удалится через 3 дня»).

**Действия:**
- **Восстановить одну** — кнопка на строке (hover). Заметка возвращается в список «Все заметки».
- **Удалить навсегда одну** — кнопка на строке (hover). Диалог подтверждения: «Удалить заметку навсегда? Это действие необратимо.»
- **Множественный выбор** — чекбоксы → панель действий: «Восстановить выбранные» / «Удалить выбранные».
- **Очистить корзину** — кнопка в верхней части вкладки. Диалог подтверждения: «Удалить все {N} заметок из корзины навсегда? Это действие необратимо.»

**Состояния:** загрузка (skeleton), пустая корзина («Корзина пуста»), ошибка загрузки.

---

### 6.5 Редактор заметки

Вкладка существует в двух режимах:

#### Режим редактирования (нет активного proposal)

- Поле заголовка (plaintext input) над редактором.
- CodeMirror 6 с live preview в тёмной теме.
- Теги заметки отображаются под заголовком — chips с кнопкой `×` для удаления и полем добавления тега (autocomplete из существующих тегов пользователя).
- Кнопка **«Прикрепить к чату»** (верхний правый угол) — прикрепляет заметку к текущему сообщению в правой панели чата (максимум 5; при превышении — tooltip «Максимум 5 заметок»).
- **Автосохранение** каждые 5 секунд при наличии изменений. Индикатор статуса (верхний правый угол): «Сохранено», «Сохраняется...», «Ошибка сохранения».
- При конфликте версий (`409 Conflict`) — уведомление: «Заметка была изменена в другом окне. Обновите страницу для загрузки актуальной версии». Автосохранение приостанавливается.

#### Режим просмотра proposal (`edit_note`)

Активируется когда у заметки есть активный proposal типа `edit_note`.

- Редактор переходит в режим diff (MDXEditor заменяется на diff-view).
- **Красная подсветка** — строки из старого текста (только чтение).
- **Зелёная подсветка** — строки из нового предложенного текста (доступны для редактирования пользователем до принятия).
- Заголовок также показывается в diff-формате если менялся.
- Нижний правый угол: одна кнопка **«Принять»** и одна **«Отклонить»** на весь proposal.
- Кнопка **«Прикрепить к чату»** остаётся доступной (верхний правый угол).
- Автосохранение **приостановлено** пока proposal активен.

### 6.6 Админ-панель

Доступна только пользователям с ролью `admin`. Кнопка на левой панели не отображается остальным пользователям.

#### Таблица пользователей

Таблица со следующими столбцами:

| Столбец        | Описание                                                              |
| -------------- | --------------------------------------------------------------------- |
| Email          | Email пользователя                                                    |
| Никнейм        | Отображаемое имя                                                      |
| Роль           | `user` / `admin` — выпадающий список для смены роли                   |
| Статус         | Активен / Заблокирован / Не подтверждён                               |
| Дата регистрации | Дата создания аккаунта (локальное время)                            |
| Действия       | Кнопки управления (см. ниже)                                          |

**Поиск:** строка фильтрации над таблицей — поиск по email и никнейму (клиентский фильтр по уже загруженным данным).

**Действия на строке:**
- **Заблокировать / Разблокировать** — переключает статус пользователя. Заблокированный пользователь не может войти. Перед блокировкой — диалог подтверждения: «Заблокировать пользователя {nickname}?»
- **Сменить роль** — выпадающий список в столбце «Роль». При смене — диалог подтверждения: «Назначить роль admin пользователю {nickname}? Пользователь получит полный доступ к администрированию.»
- **Удалить** — кнопка на строке (hover). Диалог подтверждения: «Удалить пользователя {nickname}? Все его заметки, теги и история чата будут удалены безвозвратно.» Вызывает `DELETE /api/admin/users/{id}`. При успехе строка исчезает из таблицы.
- Администратор не может изменить собственную роль, заблокировать или удалить себя (кнопки заблокированы).

**Состояния:** загрузка (skeleton-таблица), пустой список («Нет зарегистрированных пользователей»), ошибка загрузки.

#### Системная статистика

Над таблицей пользователей — панель с тремя счётчиками:

| Счётчик                     | Источник                         |
| --------------------------- | -------------------------------- |
| Пользователей               | Общее количество аккаунтов       |
| Заметок                     | Активные + soft-deleted          |
| Задач в очереди (failed)    | Embedding-задачи со статусом `failed` |

Данные загружаются при открытии вкладки. Кнопка «Обновить» в правом углу панели.

## 7. Попапы и модалки

### 7.1 Профиль

Попап открывается по кнопке «Профиль» на левой панели. Реализован как модальное окно по центру экрана с тёмным оверлеем.

#### Отображение текущих данных

Верхняя часть попапа:
- Аватар (инициал из никнейма на цветном фоне).
- Никнейм (крупный текст).
- Email пользователя (приглушённый цвет, только чтение).
- Дата регистрации (приглушённый цвет, только чтение; формат: «Зарегистрирован 12 апр. 2026»).

#### Смена никнейма

Inline-форма под текущим никнеймом:
- Поле ввода с текущим никнеймом.
- Валидация: 2–32 символа, только латиница/кириллица/цифры/`_`/`-`.
- Кнопка «Сохранить». При успехе — никнейм обновляется в попапе и в левой панели (иконка профиля).
- Ошибка 409 — «Этот никнейм уже занят».

#### Смена пароля

Форма с тремя полями:

| Поле              | Валидация                                                |
| ----------------- | -------------------------------------------------------- |
| Текущий пароль    | Обязательное                                             |
| Новый пароль      | ≥8 символов; ≥1 заглавная, ≥1 строчная, ≥1 цифра        |
| Повтор пароля     | Совпадает с новым паролем                                |

- Кнопка «Сменить пароль». При успехе — toast «Пароль успешно изменён».
- Ошибка 401 (неверный текущий пароль) — inline: «Неверный текущий пароль».

#### Удаление аккаунта

- Кнопка «Удалить аккаунт» (красная, в нижней части попапа).
- Клик открывает вложенный диалог подтверждения: «Для удаления аккаунта введите ваш пароль. Это действие необратимо — все заметки, теги и история чата будут удалены.»
- Поле пароля + кнопка «Удалить аккаунт навсегда».
- При успехе — очистка токенов, редирект на `/login`.

#### Смена языка

- Переключатель «Русский / English» (в нижней части попапа, над кнопкой «Выйти»).
- Мгновенно перерисовывает весь интерфейс на выбранном языке без перезагрузки страницы.
- Выбор сохраняется в `localeStore` и персистируется в `localStorage`.

#### Выход из аккаунта

- Кнопка «Выйти» (в нижней части попапа).
- Вызывает `POST /api/auth/logout`, очищает `authStore`, редирект на `/login`.

## 8. AI-чат (правая панель)

### 8.1 Шапка панели

Верхняя строка панели содержит (справа):
- Кнопка **«Новый чат»** — создаёт новую сессию, делает её активной.
- Кнопка **«История»** — выпадающий список предыдущих сессий. Каждая строка содержит название сессии, дату и контекстное меню (`⋯`) с действиями:
  - **Переименовать** — inline-редактирование названия прямо в строке списка. Сохранение по Enter или потере фокуса, отмена по Escape. Вызывает `PUT /api/ai/sessions/{id}` с `{ title }`.
  - **Удалить** — диалог подтверждения: «Удалить чат «{title}»? История сообщений будет удалена безвозвратно.» Вызывает `DELETE /api/ai/sessions/{id}`. Если удаляется активная сессия — автоматически создаётся новая.
  - Клик по строке (вне меню) → переключает активную сессию.

### 8.2 История сообщений

- Сообщения расположены хронологически (старые сверху, новые снизу).
- Автоскролл к новому сообщению при получении.
- Сообщения пользователя и ассистента визуально различаются (выравнивание или цвет фона пузыря).
- **Markdown-рендеринг** во всех сообщениях ассистента (заголовки, списки, код, жирный и т.д.).
- Инлайн-ссылки `[[note:uuid|Заголовок]]` парсятся и рендерятся как кликабельные ссылки → клик открывает вкладку `note:{id}` в средней панели.

### 8.3 Карточки proposals

Каждый proposal от ассистента отображается как отдельная карточка внутри сообщения.

**Содержимое карточки:**
- Тип proposal (иконка + текст: «Редактировать заметку», «Создать заметку», «Удалить заметку», «Добавить теги», «Удалить теги»).
- Название затрагиваемой заметки — **кликабельно**: открывает вкладку `note:{id}` в средней панели.
- Краткое описание изменения (из поля `data` или `summary` после применения).
- Кнопки: **«Принять»** / **«Отклонить»** — активны пока `status = pending`.
- После финализации карточка показывает итоговый `summary` и статус (применено / отклонено) вместо кнопок.

**Групповые действия (при наличии двух и более `pending` proposals в сообщении):**

Над первой карточкой proposals в сообщении отображается панель с кнопками:
- **«Принять всё»** — применяет все `pending` proposals в сообщении последовательно.
- **«Отклонить всё»** — отклоняет все `pending` proposals в сообщении.

Панель исчезает когда все proposals в сообщении финализированы.

### 8.4 Карточки pending_confirmation (bulk-операции)

- Показывают тип операции, количество затрагиваемых заметок.
- Кнопки: **«Подтвердить»** / **«Отклонить»**.
- После подтверждения — стриминг proposals по каждой заметке (карточки появляются последовательно).

### 8.5 Поле ввода

- Textarea, растущая по содержимому, с ограничением (max 4000 символов).
- Счётчик символов (отображается при приближении к лимиту: от 3500 и выше).
- Кнопка отправки (Enter — отправить, Shift+Enter — перенос строки).
- Блок прикреплённых заметок: chips с именем заметки и кнопкой `×`. Максимум 5. Заметки добавляются через кнопку «Прикрепить к чату» в редакторе (раздел 6.5).
- Во время генерации поле заблокировано; кнопка отправки трансформируется в кнопку **«Остановить»** (иконка меняется на ◼, фон становится красным). Клик вызывает `POST /api/ai/sessions/{id}/cancel`.

**Блокировка при необработанных proposals:**

Поле ввода и кнопка отправки **полностью заблокированы**, если в сообщениях текущей сессии есть хотя бы один proposal или pending_confirmation со статусом `pending`. Под полем ввода отображается пояснение: «Обработайте предложения ассистента, чтобы продолжить». Блокировка снимается автоматически когда все proposals финализированы (`applied` / `dismissed`).

### 8.6 SSE-стриминг

Ответ ассистента передаётся через SSE (`text/event-stream`). Клиент обрабатывает события:

| Событие                   | Действие клиента                                                              |
| ------------------------- | ----------------------------------------------------------------------------- |
| `ai:text_delta`           | Дописывает `delta` к тексту текущего сообщения ассистента                    |
| `ai:proposal`             | Рендерит карточку proposal (раздел 8.3)                                       |
| `ai:pending_confirmation` | Рендерит карточку bulk-операции (раздел 8.4)                                  |
| `ai:error`                | Показывает toast с описанием ошибки                                           |
| `ai:done`                 | Завершает стриминг, сохраняет итоговое сообщение, разблокирует поле ввода     |

Клиент проверяет `Content-Type` ответа перед обработкой: если `application/json` — ошибка пришла до начала стрима (400/404/409), отображается как toast.

### 8.7 Состояния

| Состояние                     | Поведение                                                                                                                        |
| ----------------------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| Ожидание ответа               | Строка состояния **«Думаю...»** с анимацией (мигающие точки) между последним сообщением пользователя и областью ответа           |
| Генерация идёт                | Текст ассистента появляется постепенно; строка «Думаю...» скрывается как только пришёл первый `ai:text_delta`; кнопка отправки трансформируется в кнопку «Остановить» (◼, красный фон) |
| Необработанные proposals      | Поле ввода заблокировано (раздел 8.5)                                                                                            |
| Ошибка генерации              | SSE `ai:error` → toast с описанием ошибки                                                                                        |
| Пустая сессия                 | Placeholder: «Задайте вопрос о ваших заметках»                                                                                   |
| Загрузка истории сессии       | Skeleton-плейсхолдеры сообщений                                                                                                  |

## 9. Стейт-менеджмент

Стейт делится на две категории: **серверный** (управляется TanStack Query) и **клиентский** (управляется Zustand).

### 9.1 TanStack Query — серверный стейт

TanStack Query используется для всех данных, которые приходят с сервера и требуют кэша, инвалидации и фоновой синхронизации.

| Query key               | Данные                                          | Инвалидация                                  |
| ----------------------- | ----------------------------------------------- | -------------------------------------------- |
| `['notes']`             | Постраничный список заметок (бесконечный скролл) | Создание, удаление, восстановление заметки   |
| `['note', id]`          | Одна заметка (заголовок, контент, теги, версия)  | Автосохранение, применение proposal          |
| `['tags']`              | Список тегов пользователя с количеством          | Создание, переименование, удаление тега      |
| `['trash']`             | Список удалённых заметок                         | Удаление, восстановление, очистка корзины    |
| `['chat-sessions']`     | Список сессий чата                               | Создание, переименование, удаление сессии    |
| `['chat-messages', id]` | История сообщений сессии                         | Завершение стрима (`ai:done`)                |
| `['search', query]`     | Результаты полнотекстового поиска                | Не инвалидируется автоматически              |
| `['rag-search', query]` | Результаты семантического поиска                 | Не инвалидируется автоматически              |
| `['admin-users']`       | Список пользователей (только admin)              | Блокировка, разблокировка, смена роли, удаление |
| `['admin-stats']`       | Системная статистика (только admin)              | Не инвалидируется автоматически; кнопка «Обновить» |

### 9.2 Zustand — клиентский стейт

Zustand хранит только состояние, которое не является данными с сервера: UI-флаги, навигацию, токены.

| Слайс        | Данные                                                                            |
| ------------ | --------------------------------------------------------------------------------- |
| `authStore`  | Access token, данные текущего пользователя (id, email, никнейм, роль), статус auth |
| `tabsStore`  | Список открытых вкладок, id активной вкладки                                      |
| `chatStore`  | id активной сессии чата; стриминговый буфер текущего сообщения (накопленный текст) |
| `uiStore`    | Правая панель открыта/закрыта; список прикреплённых заметок в поле чата            |
| `localeStore` | Активный язык (`'ru'` \| `'en'`); функция `t(key)` для получения строки; персистируется в `localStorage` |

## 10. Обработка ошибок и UX

### 10.1 Toast-уведомления

Используются для фоновых и асинхронных событий, где нет подходящего inline-контекста:

- Ошибка автосохранения заметки.
- Ошибка SSE (`ai:error`).
- Ошибки сети при мутациях (создание тега, смена роли, действия с корзиной).
- `429 Too Many Requests` на любом эндпоинте.
- Успешные действия, требующие подтверждения: «Пароль изменён», «Тег удалён», «Заметки восстановлены».

Toast отображается в правом нижнем углу экрана. Автоматически исчезает через 4 секунды. Ошибки (красные) не исчезают автоматически — требуют закрытия вручную.

### 10.2 Inline-ошибки форм

Отображаются под каждым полем при нарушении валидации:

- Валидация запускается при `blur` на поле или при попытке сабмита.
- Серверные ошибки (например, «Email занят») также отображаются inline под соответствующим полем.
- Стиль: красный текст, красная граница поля.

### 10.3 Состояния загрузки

| Контекст                             | Индикатор                                              |
| ------------------------------------ | ------------------------------------------------------ |
| Первичная загрузка списка заметок    | Skeleton-строки (3–5 заглушек)                          |
| Первичная загрузка сообщений чата    | Skeleton-пузыри                                        |
| Загрузка результатов поиска          | Skeleton в каждом разделе                              |
| Мутации (кнопки)                     | Кнопка переходит в `disabled` + inline spinner         |
| Silent refresh токена при старте     | Splash-экран (логотип по центру)                       |

### 10.4 Глобальный Error Boundary

React Error Boundary обёртывает всё приложение. При неперехваченной ошибке компонента:
- Отображается экран: «Что-то пошло не так» + кнопка «Перезагрузить страницу».
- Ошибка логируется в консоль (для отладки в dev-режиме).

### 10.5 Потеря соединения

При обнаружении offline-состояния (`navigator.onLine === false` или серии ошибок сети):
- Баннер в верхней части экрана: «Нет соединения с сервером. Проверьте подключение к интернету.»
- Баннер исчезает автоматически при восстановлении соединения.
- Автосохранение заметки приостанавливается в offline-режиме.

### 10.6 Режим read-only (второй клиент)

При подключении к системе с другого браузера или вкладки текущий клиент может перейти в режим read-only (FR-AUTH-10).

**Признак режима:** сервер возвращает `403` с кодом `readonly_session` на любой мутирующей операции.

**Визуальное отображение:**
- Фиксированный баннер в верхней части `AppLayout` (над табами): «Этот клиент работает в режиме просмотра — вы открыли приложение в другом окне. [Взять управление]»
- Кнопка «Взять управление» вызывает `POST /api/auth/takeover`; после успеха баннер исчезает и все мутирующие операции разблокируются.

**Заблокированные элементы в read-only:**
- Редактор заметки переходит в режим только чтения (поля недоступны для ввода).
- Кнопки «Создать заметку», «Удалить», «Восстановить», «Очистить корзину» — `disabled`.
- Поле ввода в чате заблокировано; кнопки proposal/confirmation — `disabled`.
- Создание/переименование/удаление тегов и сессий — `disabled`.

**Разрешено в read-only:** просмотр заметок, поиск, просмотр истории чата, навигация по вкладкам.

### 10.7 Состояние `429 Too Many Requests`

- Отображается toast: «Слишком много запросов. Подождите немного.»
- На auth-страницах — дополнительно inline под формой.
- TanStack Query не выполняет автоматический retry для статусов 4xx.

## 11. Горячие клавиши

Глобальные хоткеи, активные внутри `AppLayout` (не работают на auth-страницах).

| Сочетание          | Действие                                                                 |
| ------------------ | ------------------------------------------------------------------------ |
| `Ctrl+N`           | Создать новую заметку и открыть её в редакторе                           |
| `Ctrl+S`           | Принудительно сохранить текущую заметку (сброс таймера автосохранения)   |
| `Ctrl+W`           | Закрыть активную вкладку                                                 |
| `Ctrl+Shift+P`     | Открыть / закрыть правую панель чата                                     |
| `Ctrl+Delete`      | Мягко удалить текущую заметку (переместить в корзину) и закрыть вкладку  |

**Примечания:**
- `Ctrl+S` работает только если активная вкладка — редактор заметки (`note:{id}`); в остальных вкладках игнорируется.
- `Ctrl+Delete` аналогично работает только для вкладки редактора.
- Встроенные хоткеи CodeMirror (`Ctrl+Z`, `Ctrl+Y`) остаются в силе без дополнительной конфигурации.
- Конфликтов с браузерными дефолтами нет, кроме `Ctrl+W` (закрытие вкладки браузера) — браузерное поведение подавляется через `e.preventDefault()` только когда фокус внутри приложения.

### 12.5 Компоненты

#### Chips (теги)

- Высота: 20px. Padding: `--space-1` верт., `--space-2` гориз.
- Border-radius: `--radius-sm`. Font-size: `--font-size-sm`.
- Цвет фона: `--color-bg-input`. Цвет текста: `--color-fg-default`.
- Кнопка `×` появляется при hover на chip; удаляет тег.

#### Skeleton-плейсхолдеры

- Цвет: `--color-bg-hover` → `--color-bg-tab-bar` (анимация `pulse`).
- Строки списка: высота 56px, border-radius `--radius-md`.
- Пузыри чата: высота 40–80px, border-radius `--radius-lg`.

#### Toast-уведомления

| Тип     | Иконка   | Цвет полосы слева    | Поведение                     |
| ------- | -------- | -------------------- | ----------------------------- |
| Error   | ✕        | `--color-danger`     | Не исчезает, ручное закрытие  |
| Warning | ⚠        | `--color-warning`    | Исчезает через 6 секунд       |
| Success | ✓        | `--color-success`    | Исчезает через 4 секунды      |
| Info    | ℹ        | `--color-accent`     | Исчезает через 4 секунды      |

Позиция: правый нижний угол. Максимум 3 одновременно, старые вытесняются.

#### Диалоги подтверждения

- Ширина: 440px (фиксированная). Border-radius: `--radius-lg`.
- Структура: заголовок (bold, `--font-size-lg`) → тело (мутный текст) → строка кнопок (справа).
- Деструктивный диалог: кнопка подтверждения — `Danger`, кнопка отмены — `Ghost`.
- Backdrop: `--color-bg-overlay`, закрывается по клику вне диалога и по `Escape`.

#### Бейдж proposals

- Размер: 16×16px (минимум). Font-size: `--font-size-xs`. Border-radius: 50%.
- Цвет: bg `--color-accent`, fg white.
- Позиция: верхний правый угол иконки «Чат» в левой панели.
