Обзор

RAG (Retrieval-Augmented Generation) — подход, при котором LLM получает доступ к внешней базе знаний через механизм поиска. В Nocturn реализован двухэтапный pipeline.

Этапы

Этап 1 — Индексация: при сохранении заметки текст разбивается на чанки по 500 токенов с перекрытием 50 токенов. Для каждого чанка генерируется embedding через внешний API.

Этап 2 — Поиск: запрос пользователя преобразуется в embedding, после чего выполняется косинусный поиск по таблице note_chunks с расширением pgvector.

Очередь embedding

Генерация embedding выполняется асинхронно. Статусы задачи: pending → processing → done / failed. При недоступности сервиса задача ставится в очередь и обрабатывается позже.

Ключевые метрики качества

  • Precision@k — доля релевантных документов в top-k результатах.
  • Recall@k — покрытие всех релевантных документов в выборке.
  • MRR — средний обратный ранг первого релевантного результата.

Для быстрого запуска оценки используй:

python eval.py --dataset val --top-k 5
